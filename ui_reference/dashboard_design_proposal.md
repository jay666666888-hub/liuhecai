## 二、信息架构

### 2.1 导航结构

```
主导航 (侧边栏):
├── 📊 数据面板        → 默认首页，预测+统计混合视图
├── 🎯 预测管理        → 纯预测视图，版本选择为主
├── 📈 统计分析        → 纯统计视图，遗漏/热度为主
├── 📋 回测分析        → 回测结果展示，策略对比
├── 🤖 AI解释          → AI分析结果，预测理由
└── ⚙️ 系统设置        → 配置、主题、版本管理

底部Tab (移动端):
├── 🏠 首页
├── 🎯 预测
├── 📊 统计
└── 👤 我的
```

### 2.2 数据层级

| 层级 | 数据 | 刷新频率 |
|------|------|----------|
| L1 核心 | 当前预测、版本状态 | 手动/5分钟 |
| L2 重要 | 历史记录(最近20条)、遗漏统计 | 30秒 |
| L3 一般 | 回测结果、AI解释 | 手动/5分钟 |
| L4 扩展 | 全量历史、详细统计 | 手动 |

## 十、性能优化方案

### 10.1 首屏加载优化

```
目标: 首屏 < 2s

策略:
1. SSR/SSG (服务端预渲染)
2. 关键CSS内联
3. 图片懒加载
4. 骨架屏占位

实施:
├── 首屏只加载L1数据
├── 统计区使用骨架屏
├── 历史记录只加载前20条
└── 非关键JS延迟加载
```

### 10.2 接口优化

```
缓存策略:
├── /api/stats/all → 30s TTL
├── /api/records → 30s TTL
├── /api/versions → 5min TTL
└── /health → no cache

请求合并:
└── 统计区数据合并为 /api/stats/all 单次请求

防抖:
└── 版本切换 150ms 防抖
```

### 10.3 渲染优化

```
Virtual List:
```
