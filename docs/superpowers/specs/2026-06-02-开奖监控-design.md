# 彩票开奖监控方案

日期：2026-06-02
状态：已实施（2026-06-03 更新）

## 问题

1. cron 21:35 触发过早（开奖 21:32:32，API 尚未发布数据）
2. API 超时 3 次后回退缓存
3. 2026153 pending，无法及时验证
4. 系统单点失败：每天仅一次触发

## 方案

**新增独立监控进程 `lottery_monitor.py`**

- 21:34 启动（主 cron 前 1 分钟）
- 每 2 秒轮询 API 检查新期号
- 发现新数据立即写入缓存并执行预测流程
- 超时 120 秒自动退出监控模式

## 架构

```
21:34:00  lottery_monitor.py 启动
           ↓
     每 2 秒轮询 API
           ↓
     检查 last_processed_result_issue vs 当前期号
           ↓
    ┌─ 当前期号 <= 已处理期号 ─→ [跳过] 继续轮询
    │
    └─ 发现新期号 ─→ 写入缓存 ─→ run_live_cycle() ─→ 退出
           ↓
        超时 120s ─→ 自动退出
```

## P0 修复内容

### 1. last_issue 未更新问题

**问题**: 循环内 `last_issue` 从不更新，API 抖动会导致重复处理

**修复**: 发现新期号时立即更新 `last_issue`
```python
if current_issue != last_issue:
    last_issue = current_issue
```

### 2. 多版本判断错误

**问题**: `next_issue_prediction_exists()` 任意一个版本有预测就返回 true

**修复**: `all_versions_have_prediction()` 检查**所有**活跃版本
```python
def all_versions_have_prediction(next_issue):
    for version in active_versions:
        predictions = read_live_predictions(version, limit=10)
        if not any(p.get('issue') == next_issue for p in predictions):
            return False
    return True
```

### 3. processed_issue 机制

**持久化文件**: `data/runtime/processed_issue.json`

**判断逻辑**:
```python
if last_processed and int(current_issue) <= int(last_processed):
    log_result_skipped(current_issue, "already_processed")
    continue
```
